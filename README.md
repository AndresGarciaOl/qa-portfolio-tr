## 🚀 Sobre mí

Soy QA Engineer en formación en Tripleten, enfocado en asegurar la calidad del software a través de pruebas funcionales, validación de requisitos y documentación clara de defectos. Me interesa entender cómo funcionan los productos por dentro, detectar riesgos antes de que lleguen al usuario final y contribuir a que las aplicaciones sean estables, confiables y fáciles de usar.

Actualmente desarrollo proyectos prácticos de testing web, mobile y APIs, donde diseño y ejecuto casos de prueba, listas de verificación y pruebas de regresión, utilizando herramientas como Postman, JIRA y entornos de navegador reales. Me apoyo en técnicas como partición de equivalencias y análisis de valores límite para cubrir escenarios críticos.

Vengo de un entorno técnico-operativo, lo que me permitió desarrollar atención al detalle, disciplina en procesos y validación rigurosa de condiciones antes de dar algo por correcto. Hoy aplico esa mentalidad al QA, con un enfoque analítico, estructurado y orientado a la mejora continua.

Busco mi primera oportunidad como QA Junior o QA Manual, en equipos donde la calidad, la documentación y el trabajo colaborativo sean parte central del proceso de desarrollo.

## 📂 Proyectos Destacados

### 🚕 Urban.Routes — Testing Manual Web *(Aplicación web para reserva de taxis)*

Validación funcional de una aplicación web para la reserva de taxis, enfocada en flujos críticos de negocio, lógica de cálculo de rutas y experiencia de usuario.

📌 Durante este proyecto analicé los requisitos funcionales y diseñé casos de prueba aplicando técnicas de **partición de equivalencias** y **análisis de valores límite**, con el objetivo de validar escenarios positivos y negativos.

🎯 Ejecuté pruebas funcionales y de regresión en distintos navegadores, validando formularios, ventanas emergentes y flujos clave como selección de ruta, métodos de pago y confirmación de reserva. Los defectos identificados fueron documentados en **JIRA**, asignando severidad   y prioridad para facilitar su corrección por el equipo de desarrollo.

**🛠 Tecnologías y Herramientas:**  
Test cases · Checklists · JIRA  · Chrome DevTools

**🔍Resultados:**  
- Detección y reporte de errores funcionales y de interfaz
- Mejora de la estabilidad y consistencia de los flujos críticos
- Documentación clara y trazable de defectos

### 🛒 Urban.Grocers — Testing de API *(Aplicación web para compra de comestibles)*

Validación del backend de una aplicación web para la compra de comestibles, enfocada en la calidad, estabilidad y consistencia de los **endpoints de la API REST**.

📌 Durante este proyecto analicé la documentación técnica de la API (Swagger/Apidoc) y los requisitos funcionales de nuevas funcionalidades. Diseñé casos de prueba aplicando **partición de equivalencias** y **análisis de valores límite**, cubriendo escenarios positivos y negativos.

🎯 Ejecuté pruebas funcionales de API utilizando **Postman**, validando códigos de respuesta, estructura de JSON y manejo de errores. Los defectos detectados fueron documentados en **JIRA**, asignando severidad y prioridad para facilitar su corrección antes del despliegue.

**🛠 Tecnologías y Herramientas:**  
Postman · API Testing · Test Cases · JIRA · JSON · Swagger

**🔍 Resultados:**  
- Diseño y ejecución de casos de prueba para endpoints críticos  
- Detección y reporte de errores en respuestas y estructura de datos  
- Aseguramiento de la estabilidad del backend previo a producción  
- Documentación clara y trazable de defectos
